362 | ## 3. Cross-File Analysis Split
363 | 
364 | ### 3.1 What Aegis Does (Deterministic, Per-File)
365 | 
366 | **Aegis operates on individual files in isolation.** It uses tree-sitter AST parsing and pattern matching to detect violations that can be determined from a single file's syntax tree.
367 | 
368 | #### Examples of Aegis-Detected Violations:
369 | 
370 | 1. **AC-3 (Hardcoded Credentials)** — [`auth.js:19`](../demo/patient-portal/routes/auth.js:19)
371 |    ```javascript
372 |    if (username === 'admin' && password === 'admin') { ... }
373 |    ```
374 |    - **Detection:** AST query for `BinaryExpression` with string literal comparisons
375 |    - **No cross-file context needed:** Violation is self-contained
376 | 
377 | 2. **AC-2 (Session Timeout)** — `middleware/session.js`
378 |    ```javascript
379 |    session({ cookie: { maxAge: 86400000 } })  // 24 hours
380 |    ```
381 |    - **Detection:** AST query for `session()` call, extract `maxAge` value
382 |    - **Validation:** `maxAge > 900000` (15 minutes) → violation
383 | 
384 | 3. **EN-1 (Plaintext PHI)** — `models/Patient.js`
385 |    ```javascript
386 |    ssn: { type: DataTypes.STRING }  // No encryption
387 |    ```
388 |    - **Detection:** AST query for Sequelize model definitions, check for encryption wrappers
389 |    - **Pattern:** `DataTypes.STRING` without `get()`/`set()` hooks → violation
390 | 
391 | 4. **TR-1 (Insecure Transmission)** — `config/database.js`
392 |    ```javascript
393 |    ssl: false
394 |    ```
395 |    - **Detection:** AST query for object literals with `ssl` property
396 |    - **Validation:** `ssl === false` → violation
397 | 
398 | **Key Limitation:** Aegis **cannot** determine if a route handler actually uses authentication middleware without analyzing the middleware chain across files.
399 | 
400 | ### 3.2 What Bob Does (Contextual, Cross-File)
401 | 
402 | **Bob operates with full repository context.** It reads multiple files, understands import/export relationships, and traces data flow across module boundaries.
403 | 
404 | #### Examples of Bob-Detected Violations:
405 | 
406 | 1. **AC-1 (Missing Auth Middleware)** — [`patients.js:24`](../demo/patient-portal/routes/patients.js:24)
407 |    ```javascript
408 |    // patients.js
409 |    router.get('/:id', async (req, res) => { ... })  // No requireAuth
410 |    ```
411 |    - **Aegis sees:** Route handler definition (syntactically valid)
412 |    - **Bob sees:**
413 |      - `auth.js` exports `requireAuth` function
414 |      - `patients.js` imports `requireAuth` but doesn't use it on line 24
415 |      - Other routes in same file (`GET /`, `POST /`) **do** use `requireAuth`
416 |      - **Conclusion:** Inconsistent auth enforcement → violation
417 | 
418 | 2. **AU-1 (Missing Audit Log)** — `patients.js:24-27`
419 |    ```javascript
420 |    router.get('/:id', async (req, res) => {
421 |      const patient = await Patient.findByPk(req.params.id);
422 |      res.json(patient);  // No AuditLog.create()
423 |    })
424 |    ```
425 |    - **Aegis sees:** `Patient.findByPk()` call (syntactically valid)
426 |    - **Bob sees:**
427 |      - `GET /` route (line 9) **does** call `AuditLog.create()` after `Patient.findAll()`
428 |      - `GET /:id` route (line 24) **does not** call `AuditLog.create()`
429 |      - **Conclusion:** Inconsistent audit logging → violation
430 | 
431 | 3. **PHI Data Flow Tracing**
432 |    ```javascript
433 |    // models/Patient.js
434 |    ssn: { type: DataTypes.STRING }  // Plaintext
435 |    
436 |    // routes/patients.js
437 |    const patient = await Patient.findByPk(id);
438 |    res.json(patient);  // SSN exposed in response
439 |    ```
440 |    - **Aegis sees:** Two separate violations (plaintext storage + no encryption)
441 |    - **Bob sees:**
442 |      - SSN is stored plaintext in database
443 |      - SSN is read from database without decryption
444 |      - SSN is serialized to JSON response without redaction
445 |      - **Conclusion:** End-to-end PHI exposure → compound violation
446 | 
447 | 4. **Second-Order Violations (Import Analysis)**
448 |    ```javascript
449 |    // auth.js
450 |    exports.requireAuth = (req, res, next) => { ... }
451 |    
452 |    // patients.js
453 |    const { requireAuth } = require('./auth');  // Imported but unused
454 |    ```
455 |    - **Aegis sees:** Import statement (syntactically valid)
456 |    - **Bob sees:**
457 |      - `requireAuth` is imported but never referenced in function calls
458 |      - Other files in same directory use `requireAuth` consistently
459 |      - **Conclusion:** Dead import suggests missing auth enforcement
460 | 
461 | ### 3.3 Division of Labor (Summary Table)
462 | 
463 | | Capability | Aegis (Deterministic) | Bob (Contextual) |
464 | |------------|----------------------|------------------|
465 | | **Syntax violations** | ✅ Hardcoded credentials, weak crypto | ❌ |
466 | | **Single-file patterns** | ✅ Session timeout, plaintext storage | ❌ |
467 | | **Cross-file imports** | ❌ | ✅ Middleware chain analysis |
468 | | **Data flow tracing** | ❌ | ✅ PHI from model → route → response |
469 | | **Inconsistency detection** | ❌ | ✅ "Why does route A have auth but B doesn't?" |
470 | | **Intent reasoning** | ❌ | ✅ "Developer imported requireAuth but forgot to use it" |
471 | | **Remediation generation** | ❌ | ✅ Generate PR with fix + explanation |
472 | | **Speed** | ⚡ Milliseconds per file | 🐢 Seconds per analysis |
473 | | **False positives** | 📊 Low (pattern-based) | 📊 Medium (requires validation) |
474 | 
475 | **Design Principle:** Aegis does the heavy lifting (fast, exhaustive scanning). Bob does the nuanced reasoning (slow, context-aware analysis). Together, they provide comprehensive HIPAA compliance auditing.

621 | ## 5. Key Design Decisions and Tradeoffs
622 | 
623 | ### 5.1 Tree-Sitter vs. Regex
624 | 
625 | **Decision:** Use tree-sitter for JS/TS, regex for SQL/JSON
626 | 
627 | **Rationale:**
628 | - **Tree-sitter pros:** Accurate AST parsing, handles complex syntax, language-agnostic
629 | - **Tree-sitter cons:** Requires language grammar binaries, slower than regex
630 | - **Regex pros:** Fast, no dependencies, good for simple patterns (SQL keywords)
631 | - **Regex cons:** Fragile, can't handle nested structures
632 | 
633 | **Tradeoff:** Accuracy vs. speed. Tree-sitter for primary targets (JS/TS routes), regex for secondary targets (SQL migrations).
634 | 
635 | **Alternative Considered:** Babel parser for JS/TS
636 | - **Rejected:** Babel is JS-specific; tree-sitter supports 40+ languages (future-proof)
637 | 
638 | ### 5.2 SQLite vs. PostgreSQL
639 | 
640 | **Decision:** SQLite for findings database
641 | 
642 | **Rationale:**
643 | - **SQLite pros:** Zero-config, single file, fast for read-heavy workloads
644 | - **SQLite cons:** No concurrent writes, limited scalability
645 | - **Use case:** Local dev tool, single-user, findings are append-only
646 | 
647 | **Tradeoff:** Simplicity vs. scalability. SQLite is sufficient for hackathon scope.
648 | 
649 | **Alternative Considered:** PostgreSQL
650 | - **Rejected:** Overkill for local tool; adds deployment complexity
651 | 
652 | **Future:** If Aegis becomes multi-user SaaS, migrate to PostgreSQL with connection pooling.
653 | 
654 | ### 5.3 MCP stdio vs. TCP
655 | 
656 | **Decision:** stdio transport for MCP server
657 | 
658 | **Rationale:**
659 | - **stdio pros:** Simple IPC, no port conflicts, works with local Bob
660 | - **stdio cons:** Single client, no remote access
661 | - **Use case:** Bob runs on same machine as Aegis
662 | 
663 | **Tradeoff:** Simplicity vs. multi-client support. stdio is sufficient for local agent.
664 | 
665 | **Alternative Considered:** TCP transport on `--port`
666 | - **Rejected for v0.1:** Adds auth complexity (who can connect?)
667 | - **Future:** Implement TCP for remote Bob instances (e.g., cloud-hosted agents)
668 | 
669 | ### 5.4 Per-File vs. Cross-File Rules
670 | 
671 | **Decision:** Aegis does per-file, Bob does cross-file
672 | 
673 | **Rationale:**
674 | - **Per-file pros:** Fast, parallelizable, deterministic
675 | - **Per-file cons:** Misses import relationships, data flow
676 | - **Cross-file pros:** Comprehensive, context-aware
677 | - **Cross-file cons:** Slow, requires LLM reasoning
678 | 
679 | **Tradeoff:** Speed vs. completeness. Aegis provides fast baseline, Bob adds depth.
680 | 
681 | **Alternative Considered:** Aegis does cross-file analysis with static analysis tools (e.g., ESLint, Semgrep)
682 | - **Rejected:** Static analysis tools are rule-based, not context-aware. Bob's LLM reasoning is more flexible.
683 | 
684 | ### 5.5 PDF vs. HTML Reports
685 | 
686 | **Decision:** PDF as primary report format
687 | 
688 | **Rationale:**
689 | - **PDF pros:** Portable, printable, professional appearance
690 | - **PDF cons:** Not interactive, harder to generate
691 | - **Use case:** Compliance audits require archival-quality reports
692 | 
693 | **Tradeoff:** Portability vs. interactivity. PDF for compliance, JSON for CI/CD.
694 | 
695 | **Alternative Considered:** HTML reports
696 | - **Rejected:** HTML requires hosting or local file:// URLs (security issues)
697 | - **Future:** Add HTML export for web-based dashboards
698 | 
699 | ### 5.6 Hardcoded CFR Citations vs. External Database
